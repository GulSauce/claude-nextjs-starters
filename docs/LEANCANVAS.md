# 린 캔버스 (Lean Canvas) - 퀴즈 프롬프트 검증기

> Running Lean 권장 작성 순서: 2 → 1 → 4 → 3 → 5 → 9 → 8 → 7 → 6

---

## 2. 고객 세그먼트 (Customer Segments)

- 문서 기반 퀴즈를 생성하는 AI 프롬프트를 작성하는 **1인 개발자**
- AI를 활용해 교육 콘텐츠(퀴즈)를 제작하는 **콘텐츠 제작자**
- 메타프롬프트의 품질을 체계적으로 개선하고 싶은 사용자

**얼리어답터**: AI 프롬프트 엔지니어링에 관심이 높고, 퀴즈 생성 자동화를 이미 시도 중인 개인 개발자

---

## 1. 문제 (Problem)

1. **품질 불투명**: AI에게 퀴즈 생성을 요청하는 메타프롬프트의 품질을 객관적으로 평가할 방법이 없음
2. **개선 방향 부재**: 프롬프트를 수정해도 어떤 부분이 나아졌는지 정량적으로 비교할 수 없음
3. **모델별 최적화 부족**: GPT, Gemini, Claude 등 모델마다 최적의 프롬프트 패턴이 다르지만 가이드가 없음

**기존 대안**: 직접 퀴즈를 생성해보고 결과물로 판단 (시간 소모, 주관적 평가)

---

## 4. 해결책 (Solution)

1. **7개 루브릭 기반 자동 평가**: 명확성, 문서 기반 지시, 출력 형식, 난이도 제어, 정답 품질, 예외 처리, 모델 최적화 항목으로 100점 만점 정량 평가
2. **구체적 개선안 제공**: 항목별 피드백 + 개선된 프롬프트 자동 생성
3. **검증 이력 관리**: 반복 개선 시 과거 점수와 비교 가능한 히스토리 기능

---

## 3. 가치 제안 (Unique Value Proposition)

**메타프롬프트를 제출하면 7개 교육학 기반 루브릭으로 즉시 점수화하고, 모델별 맞춤 개선안을 제공하는 로컬 검증 도구**

한 줄 요약: _"퀴즈 생성 프롬프트의 품질을 숫자로 보여주고, 더 나은 버전을 만들어 준다"_

---

## 5. 경쟁 우위 (Unfair Advantage)

- 퀴즈 생성 도메인에 특화된 **7개 교육학 기반 루브릭** (블룸 택소노미 등 반영)
- GPT / Gemini / Claude **3대 모델별 최적화 가이드** 내장
- Claude Code 연동으로 **로컬 환경에서 외부 의존 없이** 동작
- 검증 히스토리를 통한 **반복 개선 추적** 가능

---

## 9. 채널 (Channels)

- **직접 사용**: `npm run dev`로 로컬 실행 (1인 개발 도구)
- **GitHub 저장소**: 오픈소스 공개를 통한 발견
- **개발자 커뮤니티**: 프롬프트 엔지니어링 관련 블로그/SNS 공유

---

## 8. 주요 지표 (Key Metrics)

| 지표               | 설명                                     |
| ------------------ | ---------------------------------------- |
| 검증 제출 횟수     | 도구 활용 빈도 (data/prompts/ 파일 수)   |
| 평균 점수 향상률   | 동일 사용자의 재제출 시 점수 변화 추이   |
| A/B 등급 달성 비율 | 80점 이상 프롬프트 비율 (품질 개선 효과) |
| 히스토리 재방문율  | 반복 개선 사이클 활용도                  |

---

## 7. 비용 구조 (Cost Structure)

| 항목             | 비용                                          |
| ---------------- | --------------------------------------------- |
| Claude Code 구독 | /validate 커맨드 실행을 위한 Claude 사용 비용 |
| 개발 시간        | 1인 개발자의 MVP 구축 및 유지보수 시간        |
| 인프라 비용      | **없음** (로컬 전용, 외부 서버/DB 불필요)     |

---

## 6. 수익 흐름 (Revenue Stream)

- **MVP 단계**: 수익 모델 없음 (개인 생산성 도구)
- **향후 가능성**:
  - 오픈소스 후원 (GitHub Sponsors)
  - 커스텀 루브릭 팩 판매
  - SaaS 전환 시 프리미엄 플랜 (클라우드 호스팅, 팀 공유 기능)
